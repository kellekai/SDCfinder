Leonardo Bautista-Gomez This file implements a simple memory test tool which can run as foreground process or as a daemon.

The executable can be used to:
1) Start daemon
2) Stop daemon
3) Start foreground process

Before starting a process as a daemon we first check if the daemon pid file exists and contains a valid value. The pid file name is stored in PidFileName. If such file exists we do not create a new daemon and print an error message

The client also intercepts and handles the SIGTERM signal and exists gently when such signal is received
